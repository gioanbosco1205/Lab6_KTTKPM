# Outbox Pattern Implementation Guide

## Tổng quan
Outbox Pattern đảm bảo tính nhất quán khi lưu dữ liệu và publish events. Thay vì publish event trực tiếp, chúng ta lưu event vào outbox table trong cùng transaction với dữ liệu chính, sau đó có background service xử lý publish events từ outbox.

## Schema Database

### Bảng outbox_messages
```sql
CREATE TABLE outbox_messages (
    id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    type VARCHAR(500) NOT NULL,
    json_payload TEXT NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE NOT NULL,
    is_processed BOOLEAN NOT NULL DEFAULT FALSE,
    processed_at TIMESTAMP WITH TIME ZONE NULL
);

-- Indexes
CREATE INDEX idx_outbox_type ON outbox_messages(type);
CREATE INDEX idx_outbox_is_processed ON outbox_messages(is_processed);
CREATE INDEX idx_outbox_created_at ON outbox_messages(created_at);
CREATE INDEX idx_outbox_is_processed_created_at ON outbox_messages(is_processed, created_at);
```

## Cách sử dụng

### 1. Sử dụng TransactionalEventService (Khuyến nghị)

```csharp
// Inject service
private readonly ITransactionalEventService _transactionalEventService;

// Lưu chat message và publish event trong transaction
var chatMessage = new ChatMessage 
{
    SenderId = "user1",
    SenderName = "User 1",
    Message = "Hello World",
    MessageType = MessageType.UserMessage,
    CreatedAt = DateTime.UtcNow
};

var messageCreatedEvent = new MessageCreatedEvent 
{
    MessageId = chatMessage.Id,
    SenderId = chatMessage.SenderId,
    Content = chatMessage.Message,
    CreatedAt = chatMessage.CreatedAt
};

await _transactionalEventService.SaveChatMessageAndPublishEventAsync(
    chatMessage, 
    messageCreatedEvent
);
```

### 2. Sử dụng OutboxService trực tiếp

```csharp
// Inject services
private readonly ChatDbContext _context;
private readonly IOutboxService _outboxService;

// Trong transaction
using var transaction = await _context.Database.BeginTransactionAsync();
try 
{
    // 1. Lưu dữ liệu chính
    _context.ChatMessages.Add(chatMessage);
    await _context.SaveChangesAsync();
    
    // 2. Lưu event vào outbox
    await _outboxService.AddToOutboxAsync(messageCreatedEvent);
    
    // 3. Commit transaction
    await transaction.CommitAsync();
}
catch 
{
    await transaction.RollbackAsync();
    throw;
}
```

## Background Processing

OutboxProcessorService tự động chạy mỗi 30 giây để:
1. Lấy các messages chưa được xử lý từ outbox
2. Publish chúng qua RabbitMQ
3. Đánh dấu là đã xử lý

## Lợi ích

1. **Tính nhất quán**: Đảm bảo dữ liệu và events luôn đồng bộ
2. **Độ tin cậy**: Events không bị mất khi có lỗi network
3. **Retry mechanism**: Tự động retry khi publish thất bại
4. **Performance**: Không block business logic khi publish events
5. **Monitoring**: Có thể monitor số lượng messages chưa xử lý

## Monitoring
